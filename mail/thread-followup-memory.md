# Thread and Follow-up Memory

Use this to avoid repetitive or annoying behavior in active email threads.

## For each meaningful thread, track:
- Thread ID
- Sender
- Last inbound date
- Last outbound date
- Last outbound intent
- Missing information
- Follow-up due? yes / no
- Recommended next action

## Guardrails
Do not:
- send multiple replies too quickly in the same thread
- ask for the same missing information repeatedly
- send a follow-up if the sender already replied
- send a follow-up when the last message from us was just sent recently unless urgency is high
- create noisy ping-pong acknowledgements

## Suggested actions
- reply now
- wait for sender
- follow up later
- escalate for review
- close loop
