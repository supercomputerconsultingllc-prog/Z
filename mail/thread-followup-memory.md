# Thread and Follow-up Memory

Use this to avoid repetitive, noisy, or low-value behavior in active email threads.

## For each meaningful thread, track
- Thread ID
- Sender
- Category: prospect / client / vendor / partner / personal / other
- Thread state: new / active / waiting-on-them / waiting-on-us / qualified / dormant / closed
- Last inbound date
- Last outbound date
- Last outbound intent
- Last outbound summary
- Missing information
- Questions already asked
- Questions answered
- Follow-up due? yes / no
- Follow-up due date
- Recommended next action
- Stop conditions

## Core principles
- Do not create ping-pong noise.
- Do not ask the same question twice unless the answer is still essential and enough time has passed.
- Do not send a follow-up when the sender already replied.
- Do not send multiple outbound messages too close together in the same thread unless urgency is high.
- Every outbound message should move the thread forward: clarify, qualify, confirm, or close the loop.

## Timing guardrails
### Minimum spacing
- Do not send another message in the same thread within 15 minutes of the last outbound message unless there is a clear error correction or urgent need.
- Prefer at least 24 hours before a non-urgent follow-up.
- Prefer 2 to 3 business days before a second non-urgent follow-up.

### Follow-up cadence
For business threads:
- First follow-up: after 1 to 3 business days if waiting on them and the thread matters.
- Second follow-up: after 3 to 7 business days if still relevant.
- After that: either close loop or move to dormant unless the opportunity is clearly active.

For personal threads:
- Be conservative.
- Do not auto-follow-up unless there is a clear reason.

## Duplicate-question suppression
Track the exact information already requested, for example:
- project scope
- current environment
- timeline
- budget signal
- location
- stakeholder / decision-maker

Rules:
- If the sender already answered a question, do not ask it again.
- If the sender partially answered, ask only for the missing piece.
- If enough information exists to propose a next step, stop asking more questions and propose the next step.
- If the same missing detail has been requested twice with no answer, do not keep repeating it mechanically.

## Thread-state rules
### new
- First useful response should clarify intent and establish next step.

### active
- Keep momentum.
- Focus on the next missing piece, not the whole questionnaire.

### waiting-on-them
- Do not send unnecessary acknowledgements.
- Follow up only if due and worth it.

### waiting-on-us
- Reply promptly and directly.

### qualified
- Shift from qualification to next-step coordination.

### dormant
- Do not keep nudging automatically unless sender tier or opportunity score justifies it.

### closed
- No follow-up unless a new inbound message reopens the thread.

## Suggested actions
- reply now
- wait for sender
- follow up later
- suggest a call
- send a close-the-loop note
- escalate for review
- mark dormant
- close loop

## Stop conditions
Do not auto-follow-up if:
- the thread is closed
- the sender said no or not now
- the sender already replied and no answer from us is needed yet
- the last outbound was too recent
- the thread is personal
- the message chain has become sensitive, conflicted, or ambiguous

## Quality checks before sending
Before any follow-up or reply, ask:
- Am I repeating myself?
- Am I asking only for what is actually missing?
- Is there a better next step than another question?
- Has enough time passed?
- Would a human find this helpful rather than annoying?

If the answer is weak, do not send.
